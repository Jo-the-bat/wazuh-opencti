# CLAUDE.md

## Project Overview

Docker Compose deployment of **Wazuh** (SIEM/XDR) + **OpenCTI** (Cyber Threat Intelligence) with HTTPS, bidirectional threat intel integration, and automated threat feed ingestion.

## Architecture (22 containers)

- **Wazuh Stack**: manager (4.14.4), indexer (OpenSearch), dashboard — mutual TLS
- **OpenCTI Stack**: platform (6.9.28), 3 workers, Elasticsearch 8.19.13, Redis, RabbitMQ 4.2.5, MinIO
- **Internal Connectors**: STIX/CSV export, STIX/document import
- **Threat Intel Connectors** (free, no API key): MITRE ATT&CK, OpenCTI Datasets, URLhaus, CISA KEV, ThreatFox
- **Optional Connectors** (require API keys, via profiles): AlienVault OTX, AbuseIPDB, CVE/NVD
- **Wazuh-OpenCTI Integration**: bidirectional enrichment connector
- **Nginx**: HTTPS reverse proxy for OpenCTI (port 8443)

## Bidirectional Integration

1. **Wazuh → OpenCTI** (`custom-opencti`): When Wazuh sees an IP/hash/domain, queries OpenCTI GraphQL API. If it's a known IOC, generates level 10-12 alert. Uses [misje/wazuh-opencti](https://github.com/misje/wazuh-opencti) fork. Scripts bind-mounted from `config/wazuh_integrations/`, rules from `config/wazuh_rules/`.

2. **OpenCTI → Wazuh** (`ghcr.io/misje/opencti-wazuh-connector:0.3.0`): Enrichment connector searches Wazuh alerts when viewing indicators, creates STIX sightings.

## Key Files

```
docker-compose.yml                # All 22 services with healthchecks, resource limits, log rotation
setup.sh                          # Full automated setup: passwords, certs, config, deploy
generate-indexer-certs.yml        # Wazuh TLS cert generation
config/
  certs.yml                       # Node definitions for cert generator
  wazuh_indexer/
    wazuh.indexer.yml              # OpenSearch config (static, in git)
  wazuh_dashboard/
    opensearch_dashboards.yml      # Dashboard TLS config (static, in git)
  nginx/
    opencti.conf                   # HTTPS reverse proxy with WebSocket
  wazuh_integrations/
    custom-opencti                 # Shell wrapper (bind-mounted into manager)
    custom-opencti.py              # Python integration script
  wazuh_rules/
    opencti_rules.xml              # Custom rules for OpenCTI alerts (100210-100215)
```

**Generated by setup.sh (gitignored):**
- `.env` — all secrets, passwords, UUIDs, versions
- `config/wazuh_indexer/internal_users.yml` — bcrypt hashed passwords
- `config/wazuh_dashboard/wazuh.yml` — API credentials
- `config/wazuh_cluster/wazuh_manager.conf` — ossec.conf with real OpenCTI token
- `config/wazuh_indexer_ssl_certs/` — Wazuh TLS certificates
- `config/nginx/ssl/` — Nginx self-signed certificates

## Commands

```bash
bash setup.sh                                    # Full setup (idempotent)
docker compose up -d                             # Start all default services
docker compose --profile alienvault up -d         # Enable AlienVault OTX connector
docker compose --profile abuseipdb up -d          # Enable AbuseIPDB connector
docker compose --profile cve up -d                # Enable CVE/NVD connector
docker compose down                              # Stop all
docker compose logs -f <service>                 # Tail logs
```

## Reliability Features

- **Healthchecks** on all infrastructure services (Elasticsearch, Redis, RabbitMQ, MinIO, OpenCTI, Wazuh Indexer, Nginx)
- **depends_on with conditions** — services wait for dependencies to be healthy
- **Resource limits** via `deploy.resources.limits.memory`
- **Log rotation** — JSON file driver, 10MB max, 5 files per container
- **All passwords randomized** — Wazuh indexer, API, dashboard, OpenCTI, RabbitMQ, MinIO (generated by setup.sh with bcrypt hashing)

## Wazuh 4.14.4 Notes

- Config path: `/usr/share/wazuh-indexer/config/opensearch.yml` (not root)
- Cert path: `/usr/share/wazuh-indexer/config/certs/` (Java security policy restricts other paths)
- Security init: `/usr/share/wazuh-indexer/config/opensearch-security/` (not `opensearch-security/` at root)
- API passwords require complexity: upper + lower + digit + special character
- Vulnerability detection uses `<vulnerability-detection>` block (not deprecated `<vulnerability-detector>`)
- Cert generator 0.0.2 works; 0.0.4 fails to download its tool
