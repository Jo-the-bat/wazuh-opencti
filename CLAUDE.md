# CLAUDE.md

## Project Overview

Docker Compose deployment of **Wazuh** (SIEM/XDR) + **OpenCTI** (Cyber Threat Intelligence) with HTTPS, bidirectional threat intel integration, and automated threat feed ingestion.

## Architecture (26 default + 6 optional containers)

- **Wazuh Stack**: manager (4.14.4), indexer (OpenSearch), dashboard — mutual TLS
- **OpenCTI Stack**: platform (6.9.28), 3 workers, Elasticsearch 8.19.13, Redis, RabbitMQ 4.2.5, MinIO
- **Shuffle SOAR**: backend, frontend, orborus (workflow executor), dedicated OpenSearch
- **Internal Connectors**: STIX/CSV export, STIX/document import
- **Threat Intel Connectors** (free, no API key): MITRE ATT&CK, OpenCTI Datasets, URLhaus, CISA KEV, ThreatFox, VX Vault, DISARM Framework
- **Optional Connectors** (require API keys, via profiles): AlienVault OTX, AbuseIPDB, CVE/NVD
- **Wazuh-OpenCTI Integration**: bidirectional enrichment connector
- **Nginx**: HTTPS reverse proxy for OpenCTI (port 8443) and Shuffle (port 3443)

## Bidirectional Integration

1. **Wazuh → OpenCTI** (`custom-opencti`): When Wazuh sees an IP/hash/domain, queries OpenCTI GraphQL API. If it's a known IOC, generates level 10-12 alert. Uses [misje/wazuh-opencti](https://github.com/misje/wazuh-opencti) fork. Scripts bind-mounted from `config/wazuh_integrations/`, rules from `config/wazuh_rules/`.

2. **OpenCTI → Wazuh** (`ghcr.io/misje/opencti-wazuh-connector:0.3.0`): Enrichment connector searches Wazuh alerts when viewing indicators, creates STIX sightings.

## Shuffle SOAR Integration

Shuffle provides no-code security automation workflows. After deployment:
1. Log in to Shuffle at `https://localhost:3443` with credentials from setup output
2. Create a workflow with a **Webhook** trigger to receive Wazuh alerts
3. Copy the webhook URL and uncomment the `<integration>` block in `config/wazuh_cluster/wazuh_manager.conf`
4. Restart Wazuh manager: `docker compose restart wazuh.manager`

Shuffle can also query OpenCTI for threat intelligence enrichment within workflows using the OpenCTI app (configure with `http://opencti:8080` and the admin token from `.env`).

**Note**: Shuffle requires Docker socket access (`/var/run/docker.sock`) for orborus to spawn workflow worker containers. The network is named `wazuh-opencti` explicitly so orborus-spawned containers can reach internal services.

## Web Proxy Support

To use behind a corporate proxy, copy `proxy.env.example` to `proxy.env` and set your proxy URLs:
```bash
cp proxy.env.example proxy.env
# Edit proxy.env with your proxy URLs
```
The `proxy.env` file is loaded by all internet-facing services (connectors, OpenCTI, workers, Wazuh manager). Internal services (Elasticsearch, Redis, RabbitMQ, MinIO, Wazuh indexer) do not use it. When `proxy.env` does not exist, proxy support is silently skipped.

## Security Hardening

- **Elasticsearch authentication** — `xpack.security.enabled: true` with randomized `ELASTIC_PASSWORD`
- **Redis authentication** — `requirepass` with randomized `REDIS_PASSWORD`; healthcheck uses `REDISCLI_AUTH` env (not CLI flag)
- **Wazuh connector TLS verification** — CA cert mounted, `WAZUH_OPENSEARCH_VERIFY_TLS=true`
- **Randomized Wazuh cluster key** — generated per deployment via `openssl rand -hex 16`
- **Nginx hardening** — HSTS, X-Frame-Options, X-Content-Type-Options, Referrer-Policy, `server_tokens off`, modern cipher suite (ECDHE+AESGCM/CHACHA20), SSL session cache
- **All passwords randomized** — Wazuh indexer, API, dashboard, OpenCTI, Elasticsearch, Redis, RabbitMQ, MinIO
- **Read-only config mounts** — all bind-mounted configs and certs use `:ro`
- **OpenCTI session timeout** — idle sessions expire after 30 minutes
- **Randomized health check key** — `OPENCTI_HEALTH_KEY` generated per deployment
- **Configurable TLS hostname** — `OPENCTI_HOSTNAME` in `.env` sets cert CN/SAN

## Reliability Features

- **Healthchecks** on all services — infrastructure, application, and all connectors
- **depends_on with conditions** — services wait for dependencies to be healthy
- **Resource limits** via `deploy.resources.limits.memory`; Node.js heap sized to fit container limits
- **Log rotation** — JSON file driver, 10MB default (50MB for Wazuh manager), 5 files per container
- **Pinned image versions** — all images use explicit version tags
- **setup.sh error handling** — `set -euo pipefail`, explicit checks on bcrypt hash generation, docker compose startup, security admin init, and indexer readiness

## Wazuh 4.14.4 Notes

- Config path: `/usr/share/wazuh-indexer/config/opensearch.yml` (not root)
- Cert path: `/usr/share/wazuh-indexer/config/certs/` (Java security policy restricts other paths)
- Security init: `/usr/share/wazuh-indexer/config/opensearch-security/` (not `opensearch-security/` at root)
- API passwords require complexity: upper + lower + digit + special character
- Vulnerability detection uses `<vulnerability-detection>` block (not deprecated `<vulnerability-detector>`)
- Cert generator 0.0.2 works; 0.0.4 fails to download its tool
